# v1.13.0

## Features

### $ns Mode (Ultrawork)

- **New hook:** `ns-mode.ts` — detect `$ns` keyword in user messages via `chat.message`, inject directive in `experimental.chat.system.transform`
- **Immune to duplicate injection** — `NsModeState` ensures directive is only injected once per trigger

### Session Continuation (Nonstop)

- **New hook:** `session-continuation.ts` — on `experimental.session.compacting`, checks for pending Hive tasks and builds continuation context
- **Context injection:** next pending task's name, status, and summary are appended to the system prompt so the agent auto-continues instead of stopping

### Custom Subagent Prompts

- **Schema:** `customPrompt` field added to `agent_hive.schema.json` for both built-in and custom agents
- **Types:** `customPrompt?: string` added to `AgentModelConfig` in `hive-core`
- **Runtime injection:** Scout, Forager, and Hygienic agents now append `customPrompt` to their system prompt at config-build time

## Improvements

### Language Policy (Softened)

- Changed hard `ALWAYS English` constraint → `prefer English` in 4 agent prompts (system-hook, hive, swarm, architect)
- Human-facing chat remains in user's language; thinking & subagent prompts prefer English but switching is fine

### File Permissions

- Added `read: "allow"` + `write: "allow"` to all agent definitions (zetta, architect, swarm, scout, forager, hygienic, code-reviewer, code-simplifier, micode agents)

### Model Config Cleanup

- Removed hardcoded `model` from `DEFAULT_HIVE_CONFIG` for forager-example-template
- When subagent has no `model` configured, OpenCode automatically uses the current session model
- All sanitization steps already in place to strip empty/falsy model fields

## Chores

- Bump `hive-core` to **v1.5.0**
- Bump `@hung319/opencode-hive` to **v1.13.0**
