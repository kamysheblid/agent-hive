# v1.10.12

## Bug Fixes

- **Smart Session Title** — Session title now actually updates!

### Root Cause
The smart title feature was generating titles but only logging them to console. The actual `client.session.update()` API call was never implemented (marked as "Note: Full implementation would call...").

### Fix

1. **Added `client.session.update()` call** in `index.ts`:
   ```typescript
   await client.session.update({
     path: { id: sessionID },
     body: { title },
   });
   ```

2. **Updated `hooks/smart-title.ts`** - Added optional `client` parameter to `createSmartTitleHandler()` for reusability.

3. **Enabled file-based skill loading tests** — Removed `describe.skip()` from `config-autoload-skills.test.ts`. Tests were working but mistakenly skipped.

### How It Works

When `smartTitle.enabled` is set in config:
1. First user message is captured
2. On `session.idle` event, title is generated from first message
3. Session title is updated via OpenCode API

### Configuration

```json
{
  "smartTitle": {
    "enabled": true,
    "updateThreshold": 1
  }
}
```
