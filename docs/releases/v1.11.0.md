# v1.11.0

## Vector Memory Enhancement Suite

Complete overhaul of vector memory system with sharding, quality guards, sensitive data filtering, auto-recall, auto-capture, and auto-save to project.md.

### New Features

- **рџ“¦ Sharding (Fallback Mode)** вЂ” Auto-rotate fallback shards at `maxEntriesPerShard` (default: 500). Legacy entries in root fallback dir auto-migrated to `shard_0001/` on first access.

- **вњ… Quality Guards** вЂ” Three-layer quality control before saving memories:
  - `minContentLength` (default: 10) вЂ” Reject very short content
  - `rejectRepeatedChars` вЂ” Reject spam/abuse patterns  
  - `enableDedup` вЂ” Scope-aware dedup (same content + same type+scope)
  - `enableNearDedup` вЂ” Near-duplicate detection (off by default)

- **рџ”’ Sensitive Data Filter** вЂ” Global sensitive data redaction across ALL memory write paths (vector memory tools + memory block tools + auto-capture). Patterns:
  - API keys: `sk-`, `pk-`, `sv-`, `api-` prefixes (12+ chars)
  - GitHub tokens: `ghp_`, `gho_`, `ghu_`, `ghs_`, `ghr_`, `github_pat_`
  - AWS: `AKIA`/`ASIA` access keys + `aws_secret_access_key` assignments
  - Private keys: BEGIN/END blocks (RSA, DSA, EC, ED25519, OPENSSH, PGP)
  - JWT tokens: `eyJ...` base64url triplets
  - Bearer tokens in Authorization headers
  - Database connection strings: `postgres://`, `mysql://`, `mongodb://`, etc.
  - Environment variables: `API_KEY=`, `SECRET=`, `TOKEN=`, etc.
  - Password fields: `password=`, `pwd=`, `passwd=`
  - Email addresses (configurable, off by default)
  - Custom patterns via `agent_hive.json`
  - Configurable via `vectorMemory.memoryFilter`

- **рџ”„ Auto-Recall** вЂ” Injects up to `maxMemories` (default: 5) recent vector memories into system prompt on every turn via `experimental.chat.system.transform` hook. No extra API calls вЂ” piggybacks on the system prompt build.

- **рџ“ё Auto-Capture** вЂ” Saves session snapshots (feature name, status, task progress) as vector memory during `session.compacting`. Zero-API-call вЂ” piggybacks on existing compaction event.

- **рџ’ѕ Auto-Save to project.md** вЂ” Appends session context to `.hive/memory/project/project.md` on compaction. Keeps last `maxEntries` (default: 20) entries for cross-session persistence.

- **рџЋЇ Fix Subagent Model** вЂ” All `DEFAULT_AGENT_MODELS` set to `undefined`. Agent configs now use the user's default OpenCode provider/model automatically. User can still override via `agent_hive.json в†’ agents.<name>.model`.

### Configuration

All new features are configurable via `~/.config/opencode/agent_hive.json`:

```json
{
  "vectorMemory": {
    "sharding": {
      "maxEntriesPerShard": 500
    },
    "quality": {
      "minContentLength": 10,
      "rejectRepeatedChars": true,
      "enableDedup": true,
      "enableNearDedup": false
    },
    "memoryFilter": {
      "enabled": true,
      "redactEmails": false
    },
    "autoRecall": {
      "enabled": true,
      "maxMemories": 5
    },
    "autoCapture": {
      "enabled": true,
      "includePendingTasks": true
    },
    "autoSaveProject": {
      "enabled": true,
      "maxEntries": 20
    }
  }
}
```

### Breaking Changes

None. All new features are opt-in (enabled by default, but safe defaults that won't break existing workflows).

### Bug Fixes

- Fixed `hive_vector_status` to show active shard info (index, entry count, max per shard)
- Fixed `hive_vector_add` to report quality gate rejection with actionable hints
- Fixed `containsSensitiveData` resetting `lastIndex` on global regex (caused false negatives on repeated calls)
- Fixed `(?i)` inline regex flags not supported by Bun's JavaScript engine (replaced with standard `/gi` flag)

### Internal Changes

- Added `utils/auto-recall.ts` вЂ” `formatAutoRecallInjection()`, `buildCaptureSnapshot()` (testable, no IO)
- Added `utils/sensitive-data-filter.ts` вЂ” `filterSensitiveData()`, `containsSensitiveData()` (17 patterns, 21 tests)
- Moved quality gate + sensitive filter to service layer (`vector-memory.ts`) so ALL write paths are protected
- Memory block tool (`hive_memory_set`) also applies sensitive filter before saving
- All hooks are best-effort: silently fail, never crash the plugin
