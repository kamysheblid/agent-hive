# v1.12.2

## Infrastructure

### Fix: Consolidate release workflow + Trusted Publisher (OIDC) retry

- Merged `build` → `publish-npm` → `release` into a single `release` job to eliminate artifact-passing fragility
- Added explicit `id-token: write` + `contents: write` permissions at the job level for OIDC auth
- nop: `npm install -g npm@latest` before publish to ensure latest npm OIDC support
- Switched from NPM_TOKEN to Trusted Publisher (`--provenance`) for npm authentication
