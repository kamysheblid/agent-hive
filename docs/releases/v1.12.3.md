# v1.12.3

## Fix

### Repomix tool → MCP server (non-blocking)

- **Problem:** The old repomix tool used `spawn` to run repomix as a CLI, which could still block the plugin's event loop under load, causing the host to kill the plugin process ("agent crashes with no response").
- **Solution:** Replaced the tool-based approach with a proper MCP server using `repomix --mcp`. The MCP server runs as a **separate process** with JSON-RPC communication, managed by OpenCode's MCP lifecycle.
- **Removed:** `src/tools/repomix.ts` (163 lines of spawn-based code)
- **Added:** `src/mcp/repomix.ts` — `LocalMcpConfig` running `npx --yes repomix --mcp`
- **MCP tools available:** `pack_codebase`, `pack_remote_repository`, `grep_repomix_output`

### Release workflow consolidation (v1.12.2 follow-up)

- Release config unchanged from v1.12.2 — still uses single job with Trusted Publisher (OIDC)
