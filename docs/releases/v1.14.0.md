# v1.14.0

## Features

### External Directory Permissions

- **`external_directory: "allow"`** added to 5 core agents (Zetta, Architect, Swarm, Scout, Forager)
- Agents can now access `/tmp`, `/proc`, and other paths outside the project without permission prompts
- Fixed the repetitive "allow access?" dialog when reading system directories or external paths

### explore_directory Tool

- **New tool** for structured directory tree overview with `.gitignore` filtering, binary detection, symlink handling, and root-file content preview
- Parameters: `path`, `depth` (default 3), `maxFileSize`, `showContent`
- Returns nested tree with file sizes, dir counts, and optional content preview
- Ported from opencode-codeindex tree-builder pattern

### OpenSERP MCP Server (OSS Mode)

- **New built-in MCP**: `@openserp/mcp` with 9 tools — `search` (6 engines), `mega_search`, `image_search`, `extract`, etc.
- OSS mode: no API key needed, auto-connects to self-hosted backend
- Scout gains 4 new tools in its strategy: real search engine results, cross-engine aggregation, image search, URL-to-markdown
- Complements existing Exa websearch (semantic search) with real SERP results

### 4 Micode Agent Prompt Upgrades

- **Codebase Locator**: Smart symbol search (`dora_symbol`, `dora_references`), structure preview (`look_at`), dep analysis (`dora_file`), `explore_directory` for project overview
- **Codebase Analyzer**: Dependency graph, symbol references, unused code detection (`dora_unused`), circular deps (`dora_cycles`), cross-module data flow, side-effect patterns, state mutation tracing
- **Pattern Finder**: Reference frequency analysis, import patterns, code cycles, quality metrics, variation detection across files
- **Project Initializer**: `explore_directory` overview, tech stack auto-detection (JS/TS/Python/Go/Rust/Java/Docker), parallel subagent strategy for locator+analyzer+finder, Mermaid diagrams, API surface tables, data model documentation with class diagrams

## Improvements

### Agent Permission Consistency

- All operational agents (Zetta, Architect, Swarm, Scout, Forager) now have consistent `external_directory` access
- Permission model aligned with OpenCode V2 schema

### Project-Initializer Tool Access

- `hive_task_create` + `hive_worktree_start` added to project-initializer agent
- Enables spawning subagents for parallel codebase analysis

### Worker Stalling Fix

- `hive_worktree_batch` no longer returns `terminal: true` prematurely
- Returns `openDelegations` array with `taskToolCall` info so orchestrator can spawn workers via `task()`
- Pattern: batch setup → orchestrator calls `task()` per delegation → workers execute → merge + verify

### AGENTS.md Sync

- Updated with 4 new micode agents, 18 total tools, explore_directory, OpenSERP MCP, batch workflow, and filtered tool access per agent
