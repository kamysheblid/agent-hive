# v1.14.0

## Features

### External Directory Permissions

- **`external_directory: "allow"`** added to 5 core agents (Zetta, Architect, Swarm, Scout, Forager)
- Agents can now access `/tmp`, `/proc`, and other paths outside the project without permission prompts
- Fixed the repetitive "allow access?" dialog when reading system directories or external paths

### Scout Repomix Integration

- **New tools in scout strategy:** `pack_remote_repository` for full Git repo analysis, `pack_codebase` for local directory packing
- **New "Remote Repo Strategy" section** in scout prompt — clear guidance on when to use `pack_remote_repository` vs `grep_app_searchGitHub` vs `webfetch`
- Scouts now prefer `pack_remote_repository` over `webfetch` for deep understanding of external codebases

## Improvements

### Agent Permission Consistency

- All operational agents (Zetta, Architect, Swarm, Scout, Forager) now have consistent `external_directory` access
- Permission model aligned with OpenCode V2 schema
