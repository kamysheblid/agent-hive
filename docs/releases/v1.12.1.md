# v1.12.1

## Bug Fix

### Fix 1: Correct `btca-cli` → `btca` in auto-installer (plugin crash fix)

**Root cause:** `ensureToolsInstalled()` in `tool-installer.ts` called `execSync('npm install ... btca-cli')` at top-level. Package `btca-cli` does **not exist** on npm registry (404) — the correct name is `btca`. The failed install caused a ~12-second blocking `execSync` call, freezing opencode's web server TUI and triggering the watchdog kill.

**Fix:** Corrected the package name from `btca-cli` to `btca` in both the auto-installer and tool help text.

### Fix 2: Make `ensureToolsInstalled()` truly async (prevent module-import blocking)

**Root cause:** Even with the correct package name, `execSync` blocks the event loop synchronously. Module evaluation (`import '../index.js'`) cannot resolve until the npm install completes, causing test timeouts and delayed plugin init.

**Fix:** Replaced `execSync` with `asyncExec` (`util.promisify(exec)`) so the npm install runs asynchronously in the background — plugin loads instantly, tools install in parallel.

### Internal Changes

- `utils/tool-installer.ts` — Fixed `btca-cli` → `btca` in `TOOLS` list; replaced `execSync` with `asyncExec` for non-blocking install
- `tools/btca-ask.ts` — Fixed help text references to correct package name
- `README.md` — Updated tool listing
- `package.json` — `repository.url` normalized via `npm pkg fix`
