# v1.10.11

## Bug Fixes

- **Dynamic npm prefix detection** — Doctor now dynamically gets npm prefix instead of hardcoded paths

### Root Cause
npm prefix was hardcoded to check only `/root/.local` and `/usr/local`, but npm's actual prefix could be different (e.g., `/usr`).

### Fix

1. **Added `getNpmPrefix()`** - Dynamically get npm's actual prefix:
   ```typescript
   function getNpmPrefix(): string {
     try {
       const output = execSync('npm config get prefix', { encoding: 'utf-8', timeout: 5000 });
       return output.trim();
     } catch {
       return '/usr';  // default
     }
   }
   ```

2. **Updated `checkNpmPackage`** - Now checks:
   - Current project
   - `/root/.local`
   - `/usr/local`
   - `npmPrefix` (dynamic)
   - `npmPrefix/lib/node_modules`

3. **Updated `checkCliTool`** - Now checks:
   - `/root/.local/bin`
   - `/usr/local/bin`
   - `/usr/bin`
   - `npmPrefix/bin` (dynamic)
   - Local `.bin`
